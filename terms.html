LegacyFixer

Terms of controlled access

LegacyFixer is currently provided through controlled access for selected repositories.

Controlled access status

LegacyFixer is not currently an open self-serve product. Access is reviewed and approved manually for selected public Python repositories.

Repository suitability

The first onboarding run should use a public, non-sensitive, non-production-critical repository. You must have the right to install the GitHub App on the repository you submit.

Review-first workflow

LegacyFixer does not automatically merge pull requests. Any proposed changes must be reviewed by the repository owner or maintainer before use.

No guarantee

LegacyFixer is provided for controlled evaluation. Scan results and proposed fixes may be incomplete, delayed, or unsuccessful. Repository maintainers remain responsible for reviewing dependency changes, security impact, and CI results.
