LegacyFixer

Automated Python dependency security, without PR noise

LegacyFixer scans selected Python repositories, records dependency-security results, and creates review-ready pull requests only when repository settings explicitly allow it.

Passive by default

Review-ready PRs

When enabled, fixes are proposed through isolated branches and pull requests for human review.

Repository-level control

Each repository controls whether PR creation is enabled and how many LegacyFixer PRs can be open.

Request access

LegacyFixer is currently available for selected repositories through controlled early access. It is not an open self-serve product yet.

To request access, email beta@legacyfixer.ai with one non-critical Python repository you would be comfortable reviewing in a controlled maintenance workflow.

  1. Send the GitHub repository URL for review.
  2. Start with one public, non-sensitive Python repository.
  3. Use passive monitoring for the first scan unless a controlled PR test is explicitly agreed.
  4. If selected, install the GitHub App only on the approved repository.

Current validated scope

LegacyFixer currently focuses on dependency-security maintenance for selected public Python repositories. Pull requests are optional and created only when repository-level settings allow them.

  • requirements.txt
  • selected nested requirements.txt
  • poetry.lock
  • Pipfile.lock
  • setup.cfg install_requires

For the first onboarding run, use a non-critical repository rather than production-sensitive code.
